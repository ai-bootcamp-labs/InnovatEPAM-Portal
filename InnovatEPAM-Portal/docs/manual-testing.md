# Manual Testing

The canonical manual testing guidelines for Phase 1 live in
[../specs/001-phase1-mvp/quickstart.md](../specs/001-phase1-mvp/quickstart.md).

That document defines:

- **M1** — User Story 1 (Submitter captures and tracks an idea)
- **M2** — User Story 2 (Admin reviews and decides)
- **M3** — User Story 3 (Browse and filter the catalog)
- **X1** — Responsiveness (360 / 768 / 1024 / 1440 px)
- **X2** — WCAG 2.1 AA accessibility
- **X3** — Security
- **X4** — date-fns usage

Use the **Sign-off** rubric at the end of `quickstart.md` for each release
candidate. Record results in the corresponding release ticket.
